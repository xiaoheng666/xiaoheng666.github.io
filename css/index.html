    # CSS布局方式：

    # CSS常见的布局方式包括弹性布局、固定布局、流体布局、混合布局、绝对定位布局等各种布局的优缺点

    # float浮动布局

    - 他的的兼容性比较好。
- float元素宽高和元素之间的间距都是固定的，随着页面缩小，元素会换行。（可以设置为 不换行）。
- 形成块级元素的属性
  - 需要清除浮动：  解决塌陷的问题
    - .clear{clear：both}
    - 父元素添加 overflow：hidden

    # absolute布局

    - 优点  简单直接，兼容性好。
- 缺点  脱离了文档流。

    # flex弹性布局

    - 布局简单、灵活，移动端好；
- flex布局的元素宽高和元素之间的间距是变化的，随着页面缩小，父类空白空间减少，元素能分配到的空间也减少，元素的间距也减少。
- 缺点 是ie8以下不兼容。

    # table表格布局

    - 是兼容性好，有时候布局相对简单。
- 缺点  它是对TABLE标签的不正规使用，当需要内容高度不一致时并不适应。

    # 响应式布局 （bootstarp 栅格布局）

    - 优点：
    面对不同分辨率设备灵活性强
    能够快捷解决多设备显示适应问题
- 缺点
    兼容各种设备工作量大，效率低下
    代码累赘，会出现隐藏无用的元素，加载时间加长
    一定程度上改变了网站原有的布局结构，会出现用户混淆的情况

    # Bootstrap 栅格系统 的前端开发框架首先有个打的布局容器：https://v3.bootcss.com/

    - 概念：前端框架，基于HTML CSS js  来做的一种  框架：
  - https://v3.bootcss.com/css/#overview
  - 优点：
    - 标准的html css代码规范
    - 一套简介的组件，提高开发效率，不断更新迭代
      - 2.xx版本 不维护，代码不够简洁，功能不完整
      - 3.xx版本 稳定，放弃了Ie-6   Ie-7。对IE8页面效果不太好用，
        - （偏向于移动设备WEB / 响应式布局开发；）项目；
      - 4.xx 最新版本，目前还不是很流行

    # Bootstrap 事项预定义好的这个 container这个类，为他提供两个做此用处的类：

    - 1：container 响应式布局容器  /  container-fluid 流式布局容器 （单独开发移动端）
- 2：最大屏幕：1200px         /    最大屏幕占 百分百
- container 响应式布局容器 
- bootstarp里面的container宽度是固定的 随着屏幕大小不同，container又会划分12等分
- 类后缀：                   .col-xs-份数
- .container屏幕宽度         < 768px
  - 屏幕大小  /              12 等分平均分
  - class=“col              -lg      -3”
  - 如果孩子的份数相加       等于 12  沾满整个container宽度 
  - 如果孩子的份数相加       小于 12  沾不满 整个container宽度  有空白
  - 如果孩子的份数相加       大于 12  沾满 / 并且换行 整个container宽度

    # grid布局 （ 网格系统）

    - 将页面划分成等宽得列，通过列数定义 模块化页面布局 
- 是第一个基于二维方向的布局模块。它是第一个基于网格的原生布局系统。
- 缺点  对低版本浏览器兼容性不好。
