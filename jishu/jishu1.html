# 安装，配置配置依赖的环境node + webpack +淘宝镜像

node_model:项目的依赖环境


pulic文件夹：放置静态资源（图片）。
在webpack进行，打包的时候会原封不动回打包到dist里面

src:（c程序员的源代码文件夹）：
  asste文件夹： 放置的静态资源（多个组件共用的静态资源）放置在assts 里面的静态资源，
  在webpack打包的时候，webpack会把静态资源当做一个模块，打包到js文件里面
  componennts文件夹：放置的是菲路由的全局组件

  app.vue：唯一的根组件 vue里面的组件（结构组-样式组-行为组）

  main.js：程序的入口文件，也是整个程序的最先执行的文件

# 2:项目的配置：

2-1：项目运行起来后 自动打开 ：
  packjson:
    scripts->"serve->在后面加入 --open":

    "scripts": {
      "serve": "vue-cli-service serve --open",
      "build": "vue-cli-service build",
      "lint": "vue-cli-service lint"
    },
2-2:eslint校验功能关闭,创建变量未使用的话 报错：
  在根目录下  创建一个  vue.config.js

2-3:src文件简写，配置别名：@ = src
  // ====不能在modules里面  dist里面使用
  在根目录创建一个jsgonfig.json文件
    {
      "compilerOptions":{
        "baseUrl":"./",
        "paths":{
          "@/*":[
            "src/*"
          ]
        }
      },
      "exclude":[
        "node_modules",
        "dist"
      ]
    }
