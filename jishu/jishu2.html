# 路由组件的搭建

vue-router:
    一套流程下来 如遇都是创建好的  如果没有创建手动创建：cnpm install --save vue-router
      路由组件有四个： Home  Search Login Register
      components文件夹：里面存放非路由组件（共用的全局组件）
      src/page
      page/views：放置路由组件

5-1：配置路由：
   一般放置在router文件夹里面的 
    ==index.js文件里面配置路由：
  // 配置路由的地方
    import Vue from 'vue';
    import VueRouter from 'vue-router';

    // 使用插件，你用Vue.use()方法 试用一下
    Vue.use(VueRouter);

    // 引入路由组件 首页的  搜索的  登录的 注册的   大写  
    import Home from '@/page/Home'

    // 开始配置路由
    export default new VueRouter({
      // 配置路由
      router:[
        {
          // 可以小写
          path:'/home',
          // 必须大写
          component:Home
        },
      ]
    })

# 路由配置好了 要去入口文件 main.js---结束

 5-2 总结：
  路由组件与非路由组件区别：
      1：路由组件放在page/views文件夹中  /  非路由组件一般放置在components文件夹中；
      2：路由组件需要在router文件中进行注册  /  非路由组件在使用中，一般都是以标签的形式使用;
      3：注册完成的路由，不管是啥，他们都有$router，$route 属性

      $route与$router  区别：
        $route:一般获取路由的信息【路径，quey ， params等】 ；
        $router：一般进行编程式的导航进行路由跳转【push，replace】 ；
  
  5-3 路由的跳转  
    声明式的：router-link  路由的跳转；====

# footer组件的显示隐藏：

页尾footer 在首页home 搜索search  显示的；
页尾在登录login 注册register 的时候隐藏的；
        （v-if：频繁的操作dom  耗性能，）
        （v-show：通过样式来箱显示隐藏，）
  
  6-1：我们根据组件$route获取当前路由的信息，通过路径判断foote判断显示隐藏；
  6-2：在配置路由时候。可以给路由添加路由元信息【meta：show：true/false】，配置对象，key值不能乱写；
